Contribute

Submit a dataset record for automated precheck and maintainer review.

Contributors host large files on ModelScope, Kaggle, Zenodo, or institutional storage. The catalogue uses a BLASTNet-style intake model: automated checks catch basic metadata problems, then maintainers decide whether the dataset is ready to publish.

1

Host the Data

Upload the large files to ModelScope, Kaggle, Zenodo, or trusted institutional storage. Do not place large datasets in the website repository.

2

Prepare Metadata

Add README.md, info.json, a clear data/ layout, and a reader script when the files are solver-specific binaries.

3

Request Review

Open a GitHub issue or pull request with the dataset title, description, preview image, landing page, license, citation, review track, and key metadata.

4

Precheck and Review

Automation labels the request, validates catalogue metadata in pull requests, and emails reviewers when SMTP secrets are configured. Maintainers complete the final review.

Before Review

Minimum checklist for a new record

  • Data are hosted outside this website repository.
  • The landing page or download link is public, or access restrictions are stated clearly.
  • The dataset includes README.md, info.json, and data files under a documented structure.
  • Variables, units, shapes, dtype, field locations, and interface convention are documented.
  • Grid dimensions, physical extent, coordinate order, and boundary conditions are described.
  • Custom binary files include header size, dtype, payload shape, component order, reshape order, and load_example.py.
  • License, contributors, contact, and DOI or preferred citation are explicit.
  • At least one representative preview image is provided for the catalogue page.
  • The submitter understands that automated checks are preliminary and maintainers make the final inclusion decision.

Automated Intake

The first pass checks structure, not scientific quality.

Issue Triage

Dataset submission issues receive intake labels and an automated comment so maintainers can track new requests quickly.

Metadata Validation

Pull requests run a lightweight check over datasets/*.json for required catalogue fields, access links, format information, and domain metadata.

Email Reminder

When repository SMTP secrets and reviewer emails are configured, new requests trigger an email reminder with the issue or pull request link.

Human Decision

Maintainers still review license clarity, citation, physical description, file readability, and whether the record fits the catalogue scope.

What to Send

The catalogue record should be short and verifiable.

Dataset Page Fields

Send the title, one-paragraph description, Quick Info fields, contributors, DOI, data size, access link, and a preview image.

Metadata File

Follow the data standard and include info.json in the hosted dataset package.

Submission Channel

Use a GitHub issue for a simple request. Use a pull request if you can add the dataset page, metadata file, and image yourself.

Maintainer Review

Review focuses on accessibility, metadata completeness, file readability, license clarity, and citation. It is not a scientific peer review of the paper.
