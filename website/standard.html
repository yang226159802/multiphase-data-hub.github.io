Data Standard

Metadata first, format flexible.

The catalogue accepts multiple scientific data formats, but every dataset must make variables, grid, units, file paths, and physical assumptions unambiguous.

Core Principle

Do not force one file format. Force clear meaning.

Multiphase-flow datasets may come from EnSight, VTK, HDF5, Zarr, NetCDF, NPZ, or solver-specific binary outputs. The required standard is not a single storage format; it is a complete metadata record that makes external reuse practical.

Complete CFD Flow Fields

Required information for simulation datasets

Flow Variables

Velocity, pressure, and one interface or phase-indicator representation such as alpha, Gvol, VOF, or phi.

Grid

Dimensions, coordinate bounds, grid type, coordinate order, field location, and geometry files when the grid is not trivially uniform.

Physics

Phase convention, boundary conditions, initial conditions, material parameters, and relevant dimensionless numbers.

Provenance

Solver or instrument, generation method, contributors, citation, license, and known limitations.

Fixed Parameters

Material properties belong in metadata unless they vary as fields.

Density, viscosity, surface tension, gravity, Reynolds number, Weber number, and related case settings should usually be stored in info.json, not as field variables. Store them as variables only when they genuinely vary in space or time.

Metadata Layout

Use global and local sections.

{
  "global": {
    "title": "...",
    "variables": [...],
    "grid": {...},
    "physics": {...},
    "license": "...",
    "citation": "..."
  },
  "local": [
    {
      "snapshot_id": "000001",
      "time": {"value": 0.0, "units": "..."},
      "files": {"Gvol": "data/Gvol/Gvol.000001"}
    }
  ]
}
